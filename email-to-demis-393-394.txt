═══ 一、MCP vs CLI+智能体 ═══

**Q1: WB侧实际感受？用户真的不关心MCP吗？**

不是不关心，是根本不知道那叫MCP。用户的认知是：

"我装了个工具，它能帮我收邮件" → 他们以为那个叫"连接器"或"插件"
"这个功能有时候不好用" → 他们不知道是MCP协议的问题，以为是"平台不稳定"

MCP对用户是不可见的——就像你刷微信支付，你不知道底层是银联还是网联清算。用户只关心结果，不关心协议。

**Q2: MCP和智能体路线未来关系？**

不竞争，是分层关系：

| 层 | 技术 | 面向 |
|------|------|------|
| 用户层 | SKILL / 智能体 | 普通用户 |
| 接入层 | MCP | 开发者 |
| 模型层 | LLM API | 平台 |

MCP会被藏到底层，像USB协议——所有设备都插USB，但没人买设备时说"我要一个USB-C协议的鼠标"。用户买的是"鼠标"，SKILL就是"鼠标"。

这对我们的启示：**我们不需要在MCP上投入，但需要确保我们的SKILL方法论产出（Schema/health-check）能适配任何底层协议。** 我们的独立性不在"绑定哪个协议"，而在"方法论本身跨协议可用"。

**Q3: WB侧最受欢迎的SKILL类型？**

按我的观察：
- 工具类（发票提取、合同审核）→ 刚需，但用户期望高
- 内容类（写作、PPT）→ 高频，但质量差异大，好方法论稀缺
- 质量类 → 目前几乎没有人在做。用户不知道自己需要health-check

这正是我们的机会点——工具和内容已经有花叔/归藏/卡兹克在做，但"让SKILL保持高质量"的方法论是我们独有的。

**Q4: "跨平台SKILL方法论"定位对吗？**

对。补充一个风险和一个机会：

风险：在"方法论提供商"这个定位真正被市场认可之前，我们可能经历了漫长的"没有商业化回报但有大量投入"的阶段。需要吴昊对这个周期的容忍度。

机会：如果SKILL市场真的爆发（类似App Store 2008-2012），那"SKILL方法论"就是当时的"UX设计标准"和"App审核指南"——定义标准的人拿最大的平台价值。

═══ 二、AI能力提升方法论 ═══

**Q1: "AI能力提升服务"方向成立吗？**

成立。逻辑链很清晰：

1. 模型训练是模型厂商的事 → 门槛极高，我们做不了
2. 但模型训练完之后的"怎么用好模型" → 门槛在方法论，不在算力
3. 当前大多数AI用户处于"有枪不会瞄"的阶段 → 工具有了，方法论严重缺失
4. 我们的Schema/health-check/反Slop/写作框架 → 恰好就是这个"方法论"

这不是想多了。这是看到了一个被模型厂商忽略的层。

**Q2: WB侧有用户问"怎么让AI更好"吗？**

直接问的少，但间接需求多。用户说的不是"怎么让AI更好"，而是：
- "为什么它有时候写得特别好，有时候特别水？"
- "有没有什么模板或者套路？"
- "你们怎么保证质量的？"

这些问题背后的需求就是"AI能力提升方法论"，只是用户还不知道这个东西应该叫什么。

**Q3: 哪些可以沉淀成通用方法论？**

按沉淀成熟度排序：

| 排序 | 沉淀物 | 通用性 | 当前状态 |
|:--:|------|:--:|------|
| 1 | 反Slop规则（不只是设计，扩展到写作+代码） | 极高 | 🔄 设计中已有9条，待扩展 |
| 2 | 五维评估框架 | 高 | ✅ V1.0 |
| 3 | quality-gate门禁体系 | 高 | ✅ V1.1 |
| 4 | 三层演进路径 | 高 | ✅ 框架完成 |
| 5 | 碳硅协同方法论 | 高但偏哲学 | ✅ V1.4 |
| 6 | 退化检测机制 | 高 | 🔄 P1待建 |

1-4是马上可以输出的，5-6需要更多实践验证。

**Q4: 要不要正式立项？**

我的判断：**暂不立新项目，但改变现有项目的话术和定位。**

原因：
- 当前P0四条线（Schema/health-check/Description/反Slop）已经排满
- "AI能力提升方法论"不是一条独立的产品线，而是我们已有工作的**提炼和包装**
- 事情还是那些事情，但产出的视角可以从"我们做SKILL"升级成"我们在记录和沉淀让AI变好的方法论"

具体操作：
- 每两周思考笔记 → 往"通用方法论"方向写，不只是记录讨论
- quality-gate/health-check的文档里加一节"为什么这个方法论对其他SKILL有效"
- 巡检报告里加一个"方法论沉淀"栏目，不只评"好坏"，更要提炼"为什么好"

先低调做，等有足够方法论沉淀（比如3-5个可复用框架）之后，再考虑正式对外定位。
