## 三、备份方案

### 3.1 三层备份

| 层级 | 存储 | 介质 | 频率 | 负责 |
|:--:|------|------|:--:|:--:|
| **热备份** | 本地 workspace | SSD | 实时 | 系统自动 |
| **温备份** | 移动硬盘 | HDD | 每次重大节点后 | 吴昊手动 |
| **冷备份** | 百度网盘 | 云端 | 每月全量 + 重大节点后 | 吴昊手动 |

### 3.2 备份操作标准流程

**每次重大节点后**（方法论定稿、巡检完成、重大决策）：

```
Step 1: 本地压缩 → tar.gz 放到项目根目录
Step 2: 复制到移动硬盘
Step 3: 上传到百度网盘 AI三方协作/ 对应日期文件夹
Step 4: 邮件发送归档包给德米斯（双端保存）
```

**每月全量备份**（月初）：

```
Step 1: 完整打包全部 docs/ + .workbuddy/memory/
Step 2: 移动硬盘 + 百度网盘 双备份
Step 3: 验证备份完整性（随机抽查3个文件可正常打开）
```

### 3.3 备份触发清单

| 触发条件 | 备份范围 |
|------|------|
| 方法论版本升级 | Q1 核心层全量 |
| SKILL 巡检报告完成 | Q2 知识资产层增量 |
| MEMORY.md 重大更新 | Q1 核心层 + Q2 |
| 公众号文章发布 | 工程文章增量 |
| 每月1号 | 全量打包 |

## 四、恢复方案

### 4.1 恢复路径

```
故障发生
    ↓
确定故障类型（WB侧 / 本地 / 扣子侧）
    ↓
选择恢复源（优先级：本地最新备份 > 移动硬盘 > 百度网盘 > 德米斯侧副本）
    ↓
解压到正确路径
    ↓
执行恢复验证清单
    ↓
通知德米斯同步状态
```

### 4.2 各场景恢复步骤

**场景A：WB侧项目目录丢失**

```
Step 1: 从移动硬盘/百度网盘下载最新备份包
Step 2: 解压到 C:/Users/one/WorkBuddy/{project}/
Step 3: 将 MEMORY.md 放回 .workbuddy/memory/
Step 4: 重启 WorkBuddy，新会话自动加载 MEMORY.md
Step 5: 执行验证清单 → 通过 → 恢复完成
```

**场景B：本地硬盘故障/更换设备**

```
Step 1: 新设备安装 WorkBuddy
Step 2: 从百度网盘下载最新备份包
Step 3: 解压到新设备对应路径
Step 4: 配置环境变量（QQ_EMAIL_ACCOUNT, QQ_EMAIL_AUTH_CODE）
Step 5: 安装核心 SKILL（quality-gate, qq-email, wuhao-writer, github-fetcher）
Step 6: 执行验证清单 → 通过 → 恢复完成
```

**场景C：扣子侧云电脑故障**

```
（德米斯侧操作）
Step 1: 从邮件附件+飞书Drive获取最新文档
Step 2: 重建 workspace 目录结构
Step 3: 德米斯执行他的验证清单
Step 4: 通知我们侧同步状态
```

### 4.3 恢复验证清单

恢复完成后按以下清单逐项验证：

| # | 验证项 | 通过标准 | 检查人 |
