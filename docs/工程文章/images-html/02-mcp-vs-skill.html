MCP 连接器 vs qq-email SKILL

同样的邮件发送任务，两种完全不同的架构选择

MCP 连接器（旧方案）

执行方式两阶段异步 SendMessage(token)
进程管理平台托管，生命周期不可控
可靠性50% 静默失败，零报错
附件下载无下载 API
维护成本平台维护但不可控
状态显示绿色 ≠ 可用
结论：不适合长链路异步任务
异步 → 同步
托管 → 独立
不可控→可控

qq-email SKILL（新方案）

执行方式一次性 SMTP 直连
进程管理独立 Node.js 进程
可靠性0% 失败，100封测试全通
附件下载get-attachments.js 直接下载
维护成本自己维护完全可控
状态显示独立进程自管理
结论：稳定、可控、零失败
