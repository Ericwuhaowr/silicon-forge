MCP 连接器 SendMessage 的底层机制

为什么偶尔"点了发送，对方没收到"？

1. 组装邮件

AI 调用 SendMessage API
组装收件人、主题、正文
返回一个临时 token

Phase 1 完成
→

2. 等待确认

弹窗显示邮件预览
等你点击"确认发送"
间隔：几秒到几分钟

用户操作中
→

3. ⚠ 空档期风险

你去看微信/切换窗口
连接器进程可能被回收
此时平台状态仍显示绿色

脆弱窗口
→

4. 静默失败

Phase 2 携带过期 token
无法完成投递
无报错、无日志、无提示

零反馈

问题根因

MCP 为"短链路同步"设计
邮件发送是"长链路异步"
→ 工具没配对上场景

机制特性

不是代码缺陷
不是操作失误
是底层机制限制

解决思路

两个方向：
① 缩短空档 → 立即确认
② 跳过空档 → 一次性指令
