## 二、Step 1：提示词设计规范

### 2.1 提示词四块组装法

| 模块 | 内容 | 示例 |
|------|------|------|
| **[类型]** | 明确这是公众号封面 | `WeChat public account cover, 2.35:1 horizontal landscape` |
| **[风格]** | 选定风格 + 配色 + 色号 | `Product photography, dark grey concrete wall, spotlight from above` |
| **[内容]** | 核心视觉元素 + 隐喻 | `Three frosted glass envelopes in a single row, left=red, middle=orange, right=cyan` |
| **[约束]** | 硬性禁止项 | `No Chinese text. No vertical stacking. Single horizontal row. Clean edges.` |

### 2.2 这次实战的提示词演变

| 版本 | 问题 | 修复 |
|------|------|------|
| V1（即梦） | 未指定横版，生成竖版1280×2275 | 加 `horizontal landscape` + `--ar 2.35:1` |
| V2（即梦） | 有水印、信封截图不全 | 切换工具到LOVART |
| V3（LOVART最终） | ✅ 成功 | 明确 `single row, Do NOT stack vertically, left-to-right gradient` |

### 2.3 关键教训

| # | 教训 | 严重程度 |
|------|------|:--:|
| 1 | **必须显式指定横版方向** — 不说horizontal模型默认竖版 | 🔴 致命 |
| 2 | **必须给色号** — 不给hex值颜色随机漂移 | 🟡 中等 |
| 3 | **必须限定构图** — "single row"和"do NOT stack"都需要明确 | 🟡 中等 |
| 4 | **2.35:1比例必须说明** — 很多工具不理解这个数值，需同时用文字描述 | 🟡 中等 |
| 5 | **不写中文文字是铁律** — 任何AI模型中文渲染都是灾难 | 🔴 铁律 |

## 三、Step 2：AI生图工具三层策略

### 3.1 工具分层

| 层级 | 工具 | 定位 | 优势 | 劣势 | 适用 |
|------|------|------|------|------|------|
| 🔍 探索层 | ImageGen (WB内置) / 扣子生图 | 快速验证提示词+构图方向 | 零成本、速度快、集成度高 | 质量不达公众号发布标准 | 提示词迭代验证 |
| 🏭 主力层 | 即梦 (Jimeng) | 日常封面主力 | 质量好、速度快、支持风格多样化 | 有水印需处理后移除、有积分限制 | 日常文章封面 |
| 🚀 旗舰层 | LOVART (GPT Image2) | 重点文章封面 | 质量最高、无肉眼可见水印、光影视效出色 | 成本较高（积分制）、生成等待 | 重要文章/系列首篇 |

### 3.2 选工具决策树

```
这篇文章重要吗？
  ├── 日常文章 → 即梦（主力层）
  │     └── 积分不够 → ImageGen 应急 + 后期增强
  └── 重点/首发文章 → LOVART（旗舰层）
        └── LOVART积分不够 → 即梦 + 提示词优化
```

### 3.3 各工具提示词适配差异

| 工具 | 提示词特殊要求 |
|------|------|
| ImageGen | 中文提示词可用；不需 `--ar` 参数，靠描述控制比例 |
| 即梦 | 中文提示词；必须用 `--ar 2.35:1` 显式指定比例 |
| LOVART (GPT Image2) | 英文提示词效果更好；需 "horizontal landscape" + "single row" + "Do NOT stack" 三重约束 |

## 四、Step 3：六维封面评估框架

### 4.1 六维度定义

| 维度 | 权重 | 评估标准 | 本次实战参考 |
|------|:--:|------|------|
| **1. 比例适配度** | ⭐⭐⭐⭐⭐ | 原始比例越接近2.35:1越好，减少裁剪损失 | LOVART三张中2张21:9（最佳），1张16:9（需多裁） |
| **2. 构图均衡度** | ⭐⭐⭐⭐⭐ | 主体居中、对称稳定、裁剪后主体完整 | Apple级产品摄影构图 > 科幻风 > 极简风 |
| **3. 内容隐喻准确度** | ⭐⭐⭐⭐ | 视觉能否传达文章核心叙事 | 三信封=三方案递进，光点连线增强叙事 |
| **4. 专业质感** | ⭐⭐⭐⭐ | 是否符合文章定位的视觉品质 | 技术向文章需克制高级感，避免过于花哨 |
| **5. 信息流吸睛度** | ⭐⭐⭐ | 在公众号订阅列表中的辨识度 | 色彩饱和度适中，不刺眼也不暗淡 |
| **6. 标题留白适宜度** | ⭐⭐⭐ | 画面有没有放标题文字的空间 | 上方留白干净，不被视觉元素遮挡 |

### 4.2 评估流程

```
每个候选图 → 六维打分（每维1-5⭐） → 加权排序 → AI推荐 → 吴昊终审
```

### 4.3 常见反模式

| 反模式 | 症状 | 后果 |
|------|------|------|
| 只看好看不看适配 | 选了视觉最炫的但比例不对 | 裁剪后主体被切，效果打折 |
| 色彩过于饱和 | 荧光色/高对比过度 | 公众号信息流中显得"燥"，降低专业感 |
| 过度复杂 | 一张图塞太多元素 | 缩小到列表缩略图后辨识度归零 |
| 忽略系列扩展性 | 首篇选了个性极强的风格 | 下篇无法延续风格，系列不统一 |

## 五、Step 4：裁剪处理规范

### 5.1 技术标准

| 参数 | 值 |
|------|------|
| 目标尺寸 | 900 × 383 像素 |
| 目标比例 | 2.35:1 |
| 输出格式 | JPEG（品质95%，optimize） |
| 文件大小目标 | < 300KB（微信限制） |
| 裁剪策略 | 中心裁剪（上下对称裁切） |

### 5.2 Python脚本（标准化）

```python
from PIL import Image

def crop_to_wechat_cover(src_path, dst_path):
    """
    将任意尺寸图片裁剪为微信公众号标准封面 900x383
    策略：中心裁剪 → 缩放到目标尺寸
    """
    img = Image.open(src_path)
    w, h = img.size
    target_ratio = 900 / 383  # 2.35:1

    if w / h > target_ratio:
        # 原图更宽 → 裁左右
        new_w = int(h * target_ratio)
        left = (w - new_w) // 2
        cropped = img.crop((left, 0, left + new_w, h))
    else:
        # 原图更高 → 裁上下
        new_h = int(w / target_ratio)
        top = (h - new_h) // 2
        cropped = img.crop((0, top, w, top + new_h))

    final = cropped.resize((900, 383), Image.LANCZOS)
    final.save(dst_path, quality=95, optimize=True)
    return dst_path
```

```
{文章简称}_封面_{来源}_{尺寸}.jpg

示例：
QQ邮箱上篇_LOVART封面_900x383.jpg
```

## 七、系列文章封面策略：同构不同色

### 7.1 核心原则

> **构图不变，色系替换。** 一套构图，多套配色，系列文章封面统一且各有辨识。

### 7.2 操作方式

| 步骤 | 操作 |
|------|------|
| 1 | 首篇生成封面（含完整提示词） |
| 2 | 下篇保持 `[风格]` 和 `[内容]` 不变 |
| 3 | 仅替换 `[配色]` 模块（如红→蓝，橙→绿，青→紫） |
| 4 | 裁剪使用同一脚本（参数不变） |
| 5 | 最终输出：构图一致、色系不同、风格统一 |

### 7.3 QQ邮箱系列示例

| 篇次 | 三信封配色 | 寓意 |
|------|------|------|
| 上篇 | 红 → 橙 → 青 | 问题 → 过渡 → 基础方案 |
| 下篇 | 红 → 橙 → 紫 | 问题 → 过渡 → 工程级方案 |
