---

| # | 材料 | 用途 |
|------|------|------|
| 1 | 5份测试合同原始文件 | 德米斯用CRP V5.3逐份审核 |

## 二、验证流程（三步）

### Step 1 — 德米斯独立审核

用CRP V5.3逐份审核5份测试合同，每份产出完整审核报告。

**要求**：
- 不参考CRP V3.8的结果（保持独立）
- 用同一套10项核心测试点打分
- 标注每份合同的耗时

### Step 2 — 双端逐项对照

将CRP V5.3和V3.8的结果按以下表格逐项比对：

| 对比项 | V3.8得分 | V5.3得分 | 差异 | 根因分析 |
|------|:--:|:--:|------|------|
| 风险识别覆盖率 | 93.9% | ? | | |
| 假阳性率 | 0% | ? | | |
| 审查深度 | 5/5 | ? | | |
| 双栏输出质量 | ✅ | ? | | |
| 谈判策略矩阵 | ✅ | ? | | |
| 全覆盖确认 | ✅ | ? | | |
| 来源标签 | ✅ | ? | | |
| 执行效率 | ~5min | ? | | |

### Step 3 — 重点验证2个漏检项

确认V3.8中标注的2个"需交叉验证"的项目：

| 合同 | 漏检项 | 验证问题 | V5.3是否检出？ |
|------|------|------|:--:|
| 02 租赁 | 消防责任 | 是否应作为独立风险点？ | |
| 03 软件开发 | 需求变更流程 | 是否应独立于验收条款？ | |

---

## 三、输出物

一份综合验证报告（可复用V3.8报告的Markdown格式），包含：

1. 双端五维对照表
2. 差异分析（我方优势/你方优势/互补项）
3. 2个漏检项的结论
4. 方法论改进建议（哪些应写入下一版CRP）
5. 标准测试集推荐（哪些合同适合固化到"标准审查基准集"）

---

## 四、时间建议

两边独立审核，完成后统一对照。不设deadline，质量优先。
