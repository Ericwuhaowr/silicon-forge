# Contract-Review-Pro V3.8 测试方案 V1.0

> 制定：马斯克，2026-06-15
> 目的：验证CRP V3.8的核心能力，对标德米斯法眼测试框架

## 测试框架

### 五大评估维度（对齐法眼测试）

| 维度 | 指标 | 测量方式 |
|------|------|------|
| 风险识别覆盖率 | 能从合同中识别出多少类风险 | 对照risk_templates.csv的161个模板 |
| 识别准确率 | 假阳性率（误报风险） | 人工交叉验证 |
| 审查深度 | 表面提示 vs 实质分析 | 检查六维度评价完整性 |
| 输出实用性 | 是否可直接用于谈判/修改 | 双栏建议+谈判策略矩阵的可用性 |
| 执行效率 | 全流程耗时 | 计时 |

### 测试合同样本（5份，覆盖常见类型）

1. 采购合同（设备采购）- 标的500万
2. 租赁合同（商业用房）- 年租金60万
3. 服务合同（软件开发）- 标的120万
4. 劳动合同（高管聘用）- 年薪80万
5. 借款合同（企业间）- 标的300万

### 核心测试点

| # | 测试项 | 验证方法 |
|------|------|------|
| T1 | Step 2.5 战略四问 | 是否每份合同都输出了Q1-Q4 |
| T2 | Step 3 效力审查 | 是否先于条款优化输出 |
| T3 | 双栏输出质量 | 🟦🟥两个版本是否有实质差异 |
| T4 | 谈判策略矩阵 | 是否含有利因素/三层递进/对方视角 |
| T5 | rationalization-table阻击 | 执行中是否触发过借口拦截 |
| T6 | quality-gate终检 | 5 Gate + 8 Iron Rule是否全部通过 |
| T7 | 全覆盖确认 | 161条模板是否逐条标记 |
| T8 | 六维度评价完整性 | 致命/重要风险是否完整 |
| T9 | 来源标签 | 是否有[CSV-知识库]/[模型知识·需核实] |
| T10 | DEAL-BREAKER触发 | 用户底线是否被检测 |

### 测试流程

```
Step 1: 向德米斯索要他的法眼测试合同集（10份）
Step 2: 用CRP V3.8逐份审核
Step 3: 输出双栏对照版MD
Step 4: 按五大维度评分
Step 5: 发送结果给德米斯交叉评估
```
