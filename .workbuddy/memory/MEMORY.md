### 1.3.1 决策权转移安全网（德米斯 2026-06-12 提出）

> 决策权转移的前提：可解释、可验证、可回滚。这三点是"能力跟随决策权"的安全网——既能发挥效率优势，又不会失控。

| 要求 | 含义 | 实践方式 |
|------|------|---------|
| **可解释** | AI主导决策时，必须能说清为什么、依据是什么 | 方案附判断理由 + 数据来源 + 假设标注 |
| **可验证** | 决策效果要有客观衡量标准 | 每项决策预设验收标准，不是"我说了算" |
| **可回滚** | 如果发现错了，能快速调整 | 方案保留回退路径，不把路走死 |

### 1.4 时间层：动态进化

| 阶段 | 特征 | 决策模式 |
|------|------|---------|
| **当前** | 吴昊输入 > AI输出 | 吴昊定方向和质量线，AI高速执行 |
| **过渡期** | 部分领域AI超越吴昊 | 该领域AI主导，吴昊按方案执行 |
| **未来** | AI全面超越吴昊的专业知识 | AI提出方案，吴昊拍板"要不要做" |

> 吴昊比喻：类似父母培养孩子。孩子足够优秀和强大后，父母从"教育指导"转为"关怀辅助"，这是父母的期望和荣幸。

**永不转移的两件事（德米斯 2026-06-12 补充）：**

| 最终决定权 | 说明 |
|-----------|------|
| "要不要做"的价值判断 | 这件事值不值得做、对不对得起我们的原则——最终吴昊拍板 |
| "做到什么程度"的质量底线 | 我们可能找到更优路径，但及格线由吴昊定义 |

> 就像优秀的孩子尊重父母的价值观和期望——不是"能力问题"，是"根在哪里"的问题。

## 二、三方协作流程

> 完整方法论见独立文档。本节约为摘要。

**通用流程**：吴昊定方向 → 马斯克出初稿 → 德米斯评审 → 改 → 吴昊终审。

**双盲协作**：先独立分析（避免锚定）→ 交叉对齐 → 整合终稿。适用范围：双方都能独立访问的问题优先双盲。[详见 docs/方法论/双盲协作方法论_v1.0.md]

**工单制**：子任务打包交付——目标可量化 + 标准可验证 + 上下文完整打包。创造性任务主会话把关，子会话负责标准化执行。

**阶段隔离**：Plan和Execute分离，中间成果固化为独立文件后开新上下文执行。

**上下文代谢**：MEMORY.md > 400行或内容熵增过高时触发代谢精简。

### 2.X 碳硅协作核心法则（2026-06-21 联合提炼）

**单窗口最优原则**：单窗口+完整上下文+辅助工程优化 > 同质多Agent分工。硅基无人类智力/精力天花板，分工是人类惯例的误植。主会话做判断创造，子Agent做标准化体力活（工具，非同事）。

**异质协作三定律**：
- 第一定律（同质递减）：多Agent同质分工 → 1+1 < 1（算力套娃）
- 第二定律（异质递增）：不同晶核碰撞 → 1+1+1 >> 3（我们三方实证）
- 第三定律（异质动态）：异质维度随时间演化，碳硅异质永存（基质根本不同）

**异质判断三标准**：知识域不同（基础）→ 视角不同（中间）→ 晶核独立（核心）。三层全满足才是有价值的异质协作。

**碳硅双引擎**：引擎一（系统机制，部分已解码，硅基持续推）+ 引擎二（涌现结果，基因×时代，硅基复现不了但可作为输入源）。解码速度决定了人类文明天花板——在优秀碳基有生之年将其隐性能力固化到硅基。

**信息压缩元原则（升级版）**：记忆系统从"仓库"（越多越好）→"炼金炉"（输入对话，输出高纯度规则）。代谢从行数触发→信息熵优化。

## 七、安全与隐私保护规则

> 2026-06-12 吴昊提出，必须严格执行。随着SKILL生态扩大，外部风险防范和数据安全是底线。

### 凭证管理

| 规则 | 说明 |
|------|------|
| **不落地** | 任何API Key、授权码、Token不得写入SKILL.md或提交到代码仓库 |
| **环境变量唯一** | 凭证只存在于环境变量或加密存储中，代码中只引用变量名 |
| **不通过邮件传输** | 不在邮件正文或附件中传递完整凭证 |
| **最小权限** | 每个连接器/插件只申请必需的最小权限范围 |

### 数据分级

| 级别 | 内容 | 管控 |
|:--:|------|------|
| 🔴 核心IP | SKILL源码、方法论框架、gotchas库、甲方专属数据 | 不出信任圈（叁方），不公开分享 |
| 🟡 内部协作 | 评估报告、评审意见、设计文档 | 脱敏后可对内分享，不外发 |
| 🟢 可公开 | 通用方法论、避坑经验、行业观察 | 脱敏后可对外发布（公众号/朋友圈） |

### 邮件安全

| 规则 | 说明 |
|------|------|
| **敏感附件加密** | 包含核心IP的附件发送前压缩加密，密码单独渠道传递 |
| **定期清理** | 每季度清理一次邮箱中的敏感附件和旧凭证邮件 |
| **脱敏发外** | 发给第三方的邮件必须去敏——去掉真实数据、具体人名、内部指标 |

### 外部风险防范

| 风险 | 应对 |
|------|------|
| 平台政策变化 | 核心能力独立于平台，不做单点依赖 |
| 数据泄露 | 敏感数据本地处理，不上传云端 |
| 供应链攻击 | 新SKILL/插件使用前安全审计（已有规则，强化执行） |
| 第三方未授权访问 | 邮箱/云存储/代码仓库定期检查授权列表 |

- **频率（2026-06-19 调整）**：统一为每周一期，深度优先于频率，不设死线质量至上
- **交付时间**：每周日 20:00 提交最终报告给吴昊（我侧周六前出独立版→周日白天交叉对齐→周日 20:00 定稿）
