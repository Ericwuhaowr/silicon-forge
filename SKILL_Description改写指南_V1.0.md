# SKILL Description 改写指南 V1.0

> 基于归藏"触发条件句式"方法论
> 生效日期：2026-06-12

---

## 一、核心原则

**Description 是写给 Agent 看的路由触发器，不是写给人看的宣传语。**

好的 Description 能让 Agent 在正确的时机加载正确的 SKILL，避免该加载时不加载、不该加载时乱加载。

---

## 二、句式模板

### ✅ 推荐句式：触发条件式

```
当用户需要[做什么事]时，加载此SKILL。
```

或更详细一点：

```
当用户需要[具体场景]、[具体需求]或[具体任务]时，加载此SKILL，提供[能力描述]。
```

### ❌ 避免句式：广告宣传式

```
这个SKILL可以帮你XXX，非常好用，快来试试！
```

```
强大的XXX工具，支持XXX功能，提高效率XX倍！
```

## 三、改写示例

### 示例1：发票管理SKILL

❌ 修改前（广告式）：
> "智能发票管理助手，自动识别发票信息，支持增值税核算，大幅提升财务效率。"

✅ 修改后（触发式）：
> "当用户需要管理发票、识别发票信息、进行增值税核算或处理财务票据时，加载此SKILL。"

### 示例2：PPT生成SKILL

❌ 修改前（广告式）：
> "强大的PPT生成工具，一键生成精美演示文稿，支持多种主题和版式。"

✅ 修改后（触发式）：
> "当用户需要把文章、大纲或演讲内容转成可演示的PPT/幻灯片时，加载此SKILL。"

### 示例3：代码质量检查SKILL

❌ 修改前（广告式）：
> "专业代码审查工具，智能检测代码问题，提升代码质量，减少Bug。"

✅ 修改后（触发式）：
> "当用户需要审查代码质量、检测代码问题、进行代码Review或优化代码时，加载此SKILL。"

### 示例4：内容创作SKILL

❌ 修改前（广告式）：
> "AI内容创作助手，帮助你写出高质量的文章、文案、营销内容。"

✅ 修改后（触发式）：
> "当用户需要写作、生成文案、创作内容、撰写文章或优化文字表达时，加载此SKILL。"

## 四、写作要点

1. **开头用"当"字**：明确这是一个条件判断
2. **列出3-5个触发场景**：覆盖主要使用场景，不要太窄也不要太宽
3. **用动词开头**：管理、识别、生成、审查、写作、分析...
4. **结尾用"加载此SKILL"**：明确告诉Agent要做什么
5. **控制在50字以内**：一句话说清楚，太长反而不准确

---

## 五、常见问题

### Q1：会不会太窄了，漏掉一些场景？
A：宁可窄一点，不要太宽。太宽会导致SKILL被误加载，反而影响效果。漏掉的场景可以后续根据实际使用情况补充。

### Q2：要不要写SKILL的优点/特色？
A：不要。Description是路由用的，不是卖货用的。优点特色可以写在full_description里。

### Q3：多个功能的SKILL怎么写？
A：列主要的3个功能场景，次要的不用写。比如"当用户需要管理发票、识别票据、核算增值税时"。

### Q4：需要写平台限制吗？
A：不需要。platform字段已经标记了支持的平台，Agent会判断。

## 六、检查清单

改完后对照检查：

- [ ] 开头是"当"字
- [ ] 列出了3-5个触发动词
- [ ] 结尾是"加载此SKILL"
- [ ] 没有形容词（强大的/智能的/高效的）
- [ ] 没有宣传语（提升效率/节省时间/一键生成）
- [ ] 长度在50字以内
- [ ] 是在描述"什么时候用"，不是"有多好"
